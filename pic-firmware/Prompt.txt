## Your Task

Write the **Application Layer** — the PIC16F877A firmware for **Automatic Mode only** (no ESP, no Pi, no UART communication yet). This is the foundational layer that everything else builds on top of.

### What Automatic Mode Does

1. **Sensor polling + LCD display:** Read all analog sensors (soil moisture, temperature, current) via ADC sequentially every configurable period (default 2 seconds). Read HC-SR04 for water reservoir level. Update the LCD — cycle through sensor readings on line 1 (moisture → temp → current → water level, rotating every 2 seconds), and show system status on line 2 ("Auto Mode OK", "LOCKOUT!", "Irrigating P3", etc.).

2. **Safety checks — continuous:** Run all 4 safety algorithms on every sensor read cycle. Each has TWO levels (warning then action) per course requirements:

   - **Dry-run protection:** HC-SR04 measures water level in the reservoir. Warning: water below 5cm → amber LED + buzzer beep + LCD "Low Water Warn". Action: water below 2cm → red LED + continuous buzzer + pump relay OFF + safety lockout.

   - **Overwatering protection:** Single soil moisture sensor reading above saturation threshold. Warning: moisture > 80% → amber LED + LCD "Soil Saturated". Action: moisture > 90% → red LED + pump OFF + skip entire irrigation cycle (since there's only one sensor for the whole box, if soil is saturated, no plant needs water).

   - **Overcurrent protection:** ACS712 reading. Warning: current > 3.5A → amber LED + LCD "Overcurrent Warn". Action: current > 4.5A → red LED + system-wide shutdown (pump OFF + stepper disabled) + lockout.

   - **Overheat protection:** LM35 reading. Warning: temp > 55°C → amber LED + LCD "Overtemp Warn". Action: temp > 70°C → red LED + system shutdown + lockout.

   **Lockout behavior:** Once any safety feature triggers an ACTION level, the system enters lockout. In lockout: pump is OFF, stepper is disabled (ENABLE_X HIGH), red LED on steady, buzzer on. Lockout clears ONLY when ALL sensor readings return below their WARNING thresholds with hysteresis:
   - Temperature must drop below 50°C (not 55°C) to clear overheat
   - Water level must rise above 6cm (not 5cm) to clear dry-run
   - Current must drop below 3.0A (not 3.5A) to clear overcurrent
   - Moisture must drop below 75% (not 80%) to clear overwatering

3. **Irrigation cycle (every 3 minutes):**
   - **First:** Read soil moisture. If moisture > 70% (wet enough), skip the entire cycle. Display "Soil OK - Skip" on LCD. Wait for next cycle.
   - If irrigation is needed:
     - Home the carriage (drive toward limit switch on RB3 at slow speed, back off 200 steps, set position = 0).
     - For each plant index (0 to NUM_PLANTS-1):
       - Move carriage to plant position from lookup table.
       - Wait 300ms settle time after motion stops.
       - Turn pump relay ON (RD0 active LOW) for default 5 seconds.
       - During pump-on time, poll sensors + run safety checks every ~100ms. If any safety ACTION triggers, immediately kill pump and abort the entire cycle.
       - Also poll emergency stop button during pump-on.
       - Turn pump OFF, display "Done P[i]" briefly.
     - After all plants, return home.
     - Disable stepper driver (ENABLE_X = HIGH) to save power.
   - Wait until 3-minute timer expires before next cycle.

4. **Emergency stop button:** A single momentary push button on RB4 (active LOW with pull-up).
   - **Press (falling edge detected):** Immediately kill pump (RD0 HIGH), disable stepper (RC2 HIGH), set manual lockout flag, red LED on, LCD shows "E-STOP ACTIVE".
   - **Press again (second falling edge while in manual lockout):** Clear manual lockout, return to normal automatic operation.
   - Debounce: ignore presses within 300ms of the last detected press.
   - The button is polled in the main loop AND inside the irrigation pump-on wait loop.

5. **Motor idle management:** After any movement completes and the carriage is at rest, set ENABLE_X (RC2) HIGH to disable the A4988 driver. Re-enable (RC2 LOW) before the next move.

### XC8 Compiler Specifics

- Use `#include <xc.h>` for all register definitions.
- Use `#include <stdint.h>` for `uint8_t`, `uint16_t`, `uint32_t`.
- Use `#include <stdbool.h>` for `bool`, `true`, `false`.
- Define `#define _XTAL_FREQ 20000000UL` in `config.h` (required for `__delay_ms()` and `__delay_us()`).
- **`config.h` must be included FIRST in every `.c` file** (before `<xc.h>`) so `_XTAL_FREQ` is defined before the compiler encounters any delay macros.
- Config bits in `main.c` using `#pragma config`:
  ```c
  #pragma config FOSC = HS        // High-Speed crystal 20MHz
  #pragma config WDTE = OFF       // Watchdog Timer disabled
  #pragma config PWRTE = ON       // Power-up Timer enabled
  #pragma config BOREN = ON       // Brown-out Reset enabled
  #pragma config LVP = OFF        // Low-Voltage Programming disabled (frees RB3 for I/O!)
  #pragma config CPD = OFF        // Data EEPROM code protection off
  #pragma config WRT = OFF        // Flash write protection off
  #pragma config CP = OFF         // Code protection off
  ```
  **CRITICAL: `LVP = OFF` is mandatory.** If LVP is ON, RB3 cannot be used as the limit switch input.
- Delay macros: `__delay_ms(x)`, `__delay_us(x)`.
- **XC8 free mode limitation:** `__delay_ms()` argument must be a compile-time constant. For variable delays, use a loop: `for(uint16_t i = 0; i < ms; i++) __delay_ms(1);`
- **`void main(void)` — not `int main(void)`.** XC8 for PIC expects `void main(void)`.
- **No MikroC libraries.** All peripheral drivers are register-level.
- Bit access: `PORTBbits.RB3`, `TRISCbits.TRISC0 = 0;`, etc.
- **Bank switching:** PIC16F877A registers span multiple banks. XC8 handles bank switching automatically when you use named register definitions like `TRISA`, `PORTA`, `ADCON0`, etc. Do NOT manually switch banks.
- **Data types and RAM:** PIC16F877A has only 368 bytes of RAM. Use `uint8_t` wherever possible instead of `int` (which is 16-bit on XC8). Avoid large arrays. Global variables are fine for embedded — no need for malloc.

**Read-Modify-Write hazard on PIC16F877A:** This chip has NO LAT registers. When writing individual output bits on a shared port, RMW can corrupt adjacent pins. Mitigate by using a **shadow register** approach:
  ```c
  // Example for PORTD (LCD + relay share this port)
  static uint8_t portd_shadow = 0xFF;  // initial state
  portd_shadow &= ~(1 << 0);           // clear RD0 (pump ON)
  PORTD = portd_shadow;                // write entire port at once
  ```
  Apply this consistently on PORTD and PORTB. Document the strategy in each driver file.

### ADC Conversion Details

- **ADCON1 = 0x04:** RA0, RA1, RA2 = analog. Everything else = digital. Vref+ = VDD (5V), Vref- = VSS.
- **Use 10-bit ADC readings** (combine `ADRESH << 8 | ADRESL >> 6` for left-justified, or set right-justify with `ADCON1 |= 0x80` and read `ADRESH << 8 | ADRESL`). 10-bit gives better resolution for temperature and current sensing.
- **ADC clock:** At 20MHz, use Fosc/32 → ADCS1:ADCS0 = 10 in ADCON0. TAD = 1.6µs (meets ≥1.6µs requirement).
- **Read sequence:** Select channel in ADCON0 → wait ≥20µs acquisition → set GO/DONE → poll until clear → read ADRESH:ADRESL.
- **Soil Moisture (AN0):** Capacitive sensor v1.2: dry ≈ 3V (ADC ~614), wet ≈ 1.2V (ADC ~245). Percentage: `moisture_pct = 100 - ((adc_raw - WET_ADC) * 100 / (DRY_ADC - WET_ADC))`, clamp 0–100.
- **LM35 Temperature (AN1):** 10mV/°C. `temp_C_x10 = (uint16_t)adc_raw * 500UL / 1023` (tenths of °C, e.g. 550 = 55.0°C). Use `unsigned long` cast to avoid overflow.
- **ACS712-05B Current (AN2):** Quiescent = 2.5V (ADC ~512). 185mV/A. `current_mA = abs((int16_t)adc_raw - 512) * 26`.

### HC-SR04 Ultrasonic Sensor Details

- **Trigger:** 10µs HIGH pulse on RB0, then LOW.
- **Echo:** Time the HIGH pulse on RB1 via software loop. Count iterations, multiply by known loop time to get microseconds.
- **Distance:** `distance_cm = echo_duration_us / 58`.
- **Filtering:** Take 3 readings, use median.
- **Timeout:** If RB1 stays HIGH longer than ~25ms (>4m range), break out of the loop and flag sensor error. Never hang.
- **Mounting:** Sensor faces downward into the reservoir. Shorter distance = more water. Convert to water level: `water_level_cm = TANK_HEIGHT_CM - distance_cm`. Safety checks use `water_level_cm`.

### LCD Details (HD44780-compatible, 4-bit mode)

- **Initialization sequence (mandatory, timing-critical):**
  1. Wait >40ms after power-on
  2. Send 0x30 (as nibble), wait >4.1ms
  3. Send 0x30 again, wait >100µs
  4. Send 0x30 again, wait >100µs
  5. Send 0x20 (sets 4-bit mode)
  6. Function set: 0x28 (4-bit, 2-line, 5×8 dots)
  7. Display off: 0x08
  8. Clear display: 0x01, wait >1.6ms
  9. Entry mode set: 0x06 (increment, no shift)
  10. Display on: 0x0C (display on, cursor off, blink off)
- **Write nibble:** Put upper 4 bits of data on RD4-RD7 (via shadow register!), pulse Enable (RD3) HIGH for ≥1µs then LOW, wait ≥40µs.
- **Write byte:** Send upper nibble first, then lower nibble.
- **RS pin (RD2):** LOW = command, HIGH = data.
- **16×2 display:** Line 1 starts at address 0x00, Line 2 at 0x40.
- **Provide these LCD functions:** `LCD_Init()`, `LCD_Clear()`, `LCD_SetCursor(row, col)`, `LCD_WriteChar(c)`, `LCD_WriteString(const char *str)`, `LCD_WriteNumber(int16_t num)`.
- **LCD shares PORTD with relay (RD0) and spare relay (RD1).** The shadow register for PORTD is critical here — every LCD write must preserve the relay pin states.

### Stepper Motor Details

- A4988, 1/16 microstepping (M0/M1/M2 hardwired externally).
- Step pulse:
  ```c
  PORTCbits.RC0 = 1;  __delay_us(2);
  PORTCbits.RC0 = 0;  __delay_us(2);
  ```
- Direction: RC1. HIGH = forward (away from home), LOW = reverse (toward home).
- Enable: RC2. LOW = enabled, HIGH = disabled.
- Normal speed: ~500 steps/sec (2ms/step → `__delay_us(1996)` after 4µs pulse).
- Homing speed: ~125 steps/sec (8ms/step, 1/4 normal).
- No acceleration ramp needed for this prototype.
- **Position tracking:** Maintain a `int32_t current_position` variable (in microsteps). Increment on forward step, decrement on reverse step. This is how `move_to_plant()` knows how far to move.
- **move_to_plant(uint8_t plant_index):** Calculate `target - current_position`, set direction, step that many times. Enable stepper before moving, disable after.
